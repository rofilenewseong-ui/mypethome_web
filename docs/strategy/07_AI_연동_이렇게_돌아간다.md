# AI 연동 - NanoBanana + Kling 이렇게 돌아간다
> 출력해서 펜으로 메모하세요!

---

## AI 2개를 왜 쓰나?

```
NanoBanana = 사진 → 새로운 사진 만들기  (이미지 AI)
Kling      = 사진 → 움직이는 영상 만들기 (영상 AI)

둘을 합치면:
  펫 사진 + 옷 사진
    → NanoBanana가 합성 이미지 생성
    → Kling이 그 이미지로 영상 생성
    → 홀로그램에서 재생!
```

---

## NanoBanana (이미지 AI)

### 뭘 하는 건가?
```
입력: 펫 사진 + 옷 사진 + "이런 느낌으로 만들어줘" 설명
출력: AI가 만든 새로운 이미지 1~4장
```

### 어떻게 동작하나?
```
1단계: "이거 만들어줘" 요청 보내기
   → POST https://api.nanobananaapi.ai/api/v1/nanobanana/generate
   → 보내는 것: 사진들 + 설명(프롬프트) + 몇 장 만들지
   → 받는 것: taskId (주문번호 같은 것)

2단계: "다 됐어?" 계속 물어보기 (5초마다)
   → GET .../nanobanana/record-info?taskId=주문번호
   → 받는 것: { successFlag: 0 or 1 or 2 or 3 }

   successFlag 뜻:
     0 = "아직 만드는 중이야~"  → 5초 후 다시 물어보기
     1 = "다 만들었어!"        → 결과 이미지 URL 받기! 🎉
     2 = "생성 실패했어..."     → 에러 ❌
     3 = "제너레이션 실패..."   → 에러 ❌

3단계: 결과 이미지들을 DB에 저장
```

### 필요한 설정
```
NANOBANANA_BASE_URL = API 주소 (기본값 있음)
NANOBANANA_API_KEY  = 인증 열쇠 (없으면 동작 안함!)
```

## Kling AI (영상 AI)

### 뭘 하는 건가?
```
입력: 이미지 1장 + "이렇게 움직여줘" 설명
출력: 5~10초 짜리 동영상
```

### 어떻게 동작하나?
```
1단계: 인증 토큰 만들기
   → Access Key + Secret Key로 JWT 토큰 생성
   → 이 토큰이 30분간 유효

2단계: "영상 만들어줘" 요청 보내기
   → POST https://api.klingai.com/v1/videos/image2video
   → 보내는 것:
     - model_name: "kling-v1-6" (AI 모델 이름)
     - image: 이미지 URL
     - prompt: "강아지가 주변을 둘러보고 자연스럽게 움직인다"
     - mode: "std" (표준) 또는 "pro" (고급)
     - duration: "5" 또는 "10" (초)
   → 받는 것: task_id (주문번호)

3단계: "다 됐어?" 계속 물어보기 (10초마다)
   → GET .../videos/image2video/주문번호
   → 받는 것: { task_status: "..." }

   task_status 뜻:
     "submitted"  = "접수했어"
     "queued"     = "대기 줄에 있어"
     "processing" = "만드는 중이야~"
     "succeed"    = "다 됐어!" → 영상 URL 받기! 🎉
     "failed"     = "실패..." → 에러 ❌

4단계: 영상 URL을 DB에 저장
```

### 필요한 설정
```
KLING_BASE_URL    = API 주소 (기본값 있음)
KLING_ACCESS_KEY  = 액세스 키
KLING_SECRET_KEY  = 시크릿 키
```

## AI 작업 관리자 (Job Processor)

### 왜 필요한가?
```
AI한테 "만들어줘" 하면 바로 안 됨!
→ 5~10분 기다려야 함
→ 그동안 계속 "다 됐어?" 물어봐야 함
→ 이걸 자동으로 해주는 게 Job Processor!
```

### 어떻게 동작하나?
```
NanoBanana 작업 시작
  → 5초마다 자동으로 "다 됐어?" 확인
  → 완료되면 → 결과 저장 + 확인 중단
  → 실패하면 → 에러 저장 + 확인 중단
  → 최대 10분 (120번 확인)

Kling 작업 시작
  → 10초마다 자동으로 "다 됐어?" 확인
  → 완료되면 → 영상 URL 저장 + 확인 중단
  → 실패하면 → 에러 저장 + 확인 중단
  → 최대 10분 (60번 확인)
```

### 서버가 꺼졌다 켜지면?
```
서버 시작할 때 자동으로:
  → "처리 중이던 NanoBanana 작업 있나?" 확인
  → "처리 중이던 Kling 작업 있나?" 확인
  → 있으면 다시 확인 시작! (재개)
```

## AI 프로필 생성 전체 과정 (흐름도)

```
[유저]
  │
  ├─→ 1. 옷 사진 업로드
  │      → profile 상태: "draft" → "clothes_uploaded"
  │
  ├─→ 2. "이미지 생성해줘" 버튼
  │      → NanoBanana에 요청
  │      → profile 상태: "images_generating"
  │      │
  │      ├─→ 5초마다 확인...
  │      │
  │      └─→ 완료! AI 이미지 3~4장 도착
  │           → profile 상태: "images_ready"
  │
  ├─→ 3. 마음에 드는 이미지 1장 선택
  │      → profile 상태: "image_selected"
  │
  ├─→ 4. "영상 만들어줘" 버튼
  │      → Kling에 요청
  │      → profile 상태: "video_generating"
  │      │
  │      ├─→ 10초마다 확인...
  │      │
  │      └─→ 완료! 영상 URL 도착
  │           → profile 상태: "active" ← 완성! 🎉
  │
  └─→ 5. 홀로그램 플레이어에서 재생!
```

## 크레딧 시스템 총정리

### 크레딧을 얻는 방법
```
버전1 MVP:  주문번호 인증 → +120
버전1 모바일: 초기 120 + 직접 충전
버전2 V1:   로그인하면 제작권 3회
버전3:      구매코드 인증 → +120
```

### 크레딧을 쓰는 곳
```
버전1 MVP:  영상 1개 = 20 크레딧
버전1 모바일: 펫추가 = 1, 영상세트 = 20
버전2 V1:   영상세트 = 제작권 1회
버전3:      영상 3개 = 90 크레딧 (30 x 3)
```

## 실행하는 방법

### 버전1 (MVP)
```
1. 터미널에서 해당 폴더로 이동
2. npm install     ← 처음 한 번만
3. npm run dev     ← 서버 시작
4. 브라우저에서 http://localhost:5050 접속
```

### 버전2 (V1)
```
1. 터미널에서 해당 폴더로 이동
2. npm install     ← 처음 한 번만
3. npm run dev     ← 서버 시작
4. 브라우저에서 http://localhost:3000 접속
```

### 버전3 (petmorial)
```
1. 터미널에서 해당 폴더로 이동
2. npm run install:all  ← 서버+클라이언트 둘 다 설치
3. npm run dev          ← 서버+클라이언트 동시 시작
4. 서버: http://localhost:3001
5. 화면: http://localhost:5173
```
