```
버전1,2 = 연습용 (가짜 데이터, 임시 저장)
버전3   = 실제 서비스용!

         ✅ 관리자 페이지
```

## 유저가 사용하는 전체 흐름

### Step 1: 로그인
```
[로그인 화면]
  이메일 입력 → 로그인 버튼 클릭
  → 서버에서 출입증(JWT) 발급 (30일간 유효)
  → 홈 화면으로 이동
```

### Step 2: 구매 코드 입력
```
[코드 인증 화면]
  구매 코드 입력 → 인증 버튼
  → 코드가 유효한지 확인
  → 구매자 이메일과 내 이메일이 같은지 확인
  → 통과하면 → 크레딧 120개 충전!
```

### Step 3: 펫 등록
```
[펫 관리 → 펫 등록]
  이름, 생일, 기일, 좋아한 것, 사진 입력
  → 저장하면 펫 목록에 추가
  → 채팅방도 자동으로 만들어짐!
```

### Step 4: AI 영상 생성 (2가지 방법)

**방법 A: 간단한 방법 (레거시)**
```
[영상 생성 화면]
  사진 3~4장 업로드 → "영상 생성" 버튼
  → 크레딧 90 차감 (30 x 3개)
  → 서버에서 Mock 영상 3개 생성 (3~8초)
```

**방법 B: AI 파이프라인 (프로필 방식) ← 이게 진짜!**
```
[프로필 생성 화면]

Step 4-1: 옷 사진 업로드
  → 내 펫이 입을 옷 사진을 올림

Step 4-2: NanoBanana AI가 이미지 생성
  → 펫 사진 + 옷 사진 → AI가 합성!
  → 3~4장의 AI 이미지가 나옴
  → (5초마다 "다 됐나?" 확인, 최대 10분)

Step 4-3: 마음에 드는 이미지 선택

Step 4-4: Kling AI가 영상 생성
  → 선택한 이미지 → 움직이는 영상으로!
  → (10초마다 "다 됐나?" 확인, 최대 10분)

Step 4-5: 완성!
  → 프로필이 "active" 상태가 됨
  → 홀로그램 플레이어에서 재생 가능
```

### Step 5: 홀로그램 재생
```
[플레이어 화면] (전체화면)
  기본 영상 자동 재생
  좌측 터치 → 왼쪽 인터랙션 영상
  우측 터치 → 오른쪽 인터랙션 영상
  → 몇 초 후 기본으로 복귀
```

### Step 6: 채팅
```
[채팅방 화면]
  메시지 입력 → 전송
  → 내 메시지 바로 표시
  → 5~30초 후 펫이 이모지 답장 (서버에서 랜덤)
  → 답장 오면 알림 뱃지 표시
```

## 데이터베이스 (DB) - 서류 캐비넷 비유

> 모든 정보가 `petmorial.db` 파일 하나에 저장됨

### 서랍 1: 사람 관련
```
📁 users (유저)
   이름, 이메일, 크레딧 잔액

📁 purchase_codes (구매 코드)
   코드번호, 구매자 이메일, 사용여부, 크레딧 120

📁 credit_ledger (크레딧 기록)
   누가, 언제, 얼마나, 왜 (충전? 사용? 환불?)
```

### 서랍 2: 펫 관련
```
📁 pets (펫 정보)
   이름, 생일, 기일, 좋아한것, 프로필 사진

📁 media_photos (사진들)
   프로필 사진 or 영상용 원본 사진
```

### 서랍 3: 영상 관련
```
📁 video_jobs (영상 생성 작업)
   상태: 대기중 → 진행중 → 성공/실패

📁 videos (완성된 영상)
   영상 URL, 길이, 상태

📁 playback_settings (재생 설정)
   기본 영상, 좌측 영상, 우측 영상
```

### 서랍 4: AI 파이프라인
```
📁 profiles (AI 프로필)
   상태: 초안 → 옷업로드 → 이미지생성중 → 이미지완료
         → 이미지선택 → 영상생성중 → 활성화!

📁 nanobanana_tasks (이미지 AI 작업)
   상태: 대기 → 제출 → 처리중 → 완료/실패

📁 kling_tasks (영상 AI 작업)
   상태: 대기 → 제출 → 처리중 → 성공/실패

📁 base_videos (기본 영상)
📁 motions (모션 영상 - 좌/우 인터랙션)
```

### 서랍 5: 채팅
```
📁 chat_threads (채팅방)
   유저-펫 1:1 매칭, 안읽은 수

📁 messages (메시지)
   보낸사람(유저/펫), 내용, 상태(전송/읽음/예약)
```
