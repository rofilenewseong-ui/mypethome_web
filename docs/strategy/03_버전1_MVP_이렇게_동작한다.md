## PC 화면 (index.html + app.js) - 4개 구역

### 구역 1: 로그인 + 구매인증

```
[Google 로그인 버튼]  ← 누르면 Google 계정으로 로그인

[주문번호 입력칸] [주문 인증 버튼]
  ← 자사몰에서 산 사람만 쓸 수 있게
  ← 인증 성공하면 크레딧 120개 지급

[현재 크레딧: 120]
  ← 영상 1번 만들 때마다 20개 차감
```

**이 구역에서 일어나는 일:**
```
유저가 Google 로그인 클릭
  → Google에서 "이 사람 맞아요" 확인
  → 우리 서버에서 "출입증(JWT)" 발급
  → 브라우저에 출입증 저장 (다음에 자동 로그인)

유저가 주문번호 입력 후 "주문 인증" 클릭
  → 서버가 등록된 주문번호인지 확인
  → 맞으면 → 크레딧 120 충전!
  → 틀리면 → "인증할 수 없습니다" 에러
```

### 구역 2: 사진 업로드 + 영상 생성

```
(강아지) (고양이)  ← 동물 종류 선택

[사진 업로드 (최대 12장)]
[원하는 영상 분위기 입력]
[나노바나나 + 클링으로 영상 생성] 버튼
```

**이 구역에서 일어나는 일:**
```
유저가 사진 선택
  → "선택됨: 3장" 표시

유저가 "영상 생성" 버튼 클릭
  → 로그인 했나? (안했으면 "먼저 로그인해주세요")
  → 주문 인증했나? (안했으면 "먼저 주문 인증하세요")
  → 사진 선택했나? (안했으면 "사진을 먼저 업로드해주세요")
  → 다 통과하면:
    → 사진을 서버에 업로드
    → 서버가 AI한테 영상 만들어달라고 요청
    → 크레딧 20 차감
    → 작업 목록에 "생성 중" 표시
    → 완료되면 → 홀로그램 플레이어에 영상 표시
```

### 구역 3: 홀로그램 플레이어

```
[◀ 이전]  [10cm x 10cm 영상 화면]  [다음 ▶]
```

**이 구역에서 일어나는 일:**
```
영상이 생성되면 → 가운데 화면에 자동 재생
"이전/다음" 버튼으로 여러 영상 전환 (순환)
```

### 구역 4: 이모지 메신저

```
[채팅 화면]
  나: "오늘도 보고 싶어."
  강아지: 🐶💛✨

[메시지 입력]  [전송]
```

**이 구역에서 일어나는 일:**
```
유저가 메시지 입력 후 전송 (버튼 or Enter)
  → 내 말풍선 표시
  → 서버에 메시지 보냄
  → 서버가 랜덤 이모지 2~3개 조합해서 답장
  → 강아지 말풍선 표시
```

## 모바일 화면 (mobile.html + mobile.js) - 5개 화면

### 화면 전환 방식
```
[홈 화면]에서 아이콘 터치 → 해당 화면으로 이동
각 화면 왼쪽 위에 "← 홈" 버튼으로 돌아오기
```

### 화면 1: 홈

```
┌─────────────────────┐
│  Always Here         │
│  Heaven Box          │
│  언제나 여기에서,     │
│  너를 기억할게.       │
├─────────────────────┤
│  우리 아이들          │
│  [뽀삐] [콩이] ...   │ ← 등록한 펫 아이콘들 (터치하면 선택)
├─────────────────────┤
│  🎞 영상 만들기  💬 펫 메신저  │ ← 터치하면 해당 화면으로
│  👤 마이페이지               │
└─────────────────────┘
```

### 화면 2: 영상 만들기 (videoView)

```
[← 홈]  홀로그램 영상

[선택한 펫 프로필]

┌─────────────────────┐
│  [좌측]  [영상 화면]  [우측]  │ ← 좌측 터치: 쓰다듬기(B)
│         홀로그램             │ ← 우측 터치: 간식주기(C)
│                             │ ← 2.8초 후 기본(A)으로 복귀
└─────────────────────┘

[사진 업로드 (3~4장)]
[3분할 영상 세트 만들기 (20 크레딧)]
```

**영상 3종 세트란?**
```
A = 기본 영상 (idle)     ← 가만히 있는 모습
B = 쓰다듬기 영상 (pet)   ← 좌측 터치 시 재생
C = 간식주기 영상 (feed)  ← 우측 터치 시 재생
```

### 화면 3: 펫 메신저 (chatView)

```
카카오톡 스타일 채팅 화면
  나 (노란 말풍선): "보고싶다"
  강아지 (하얀 말풍선): 🐶💛

특별한 점: 보내면 바로 답장 안 옴!
→ 4~5시간 후 랜덤하게 "읽음" 표시 + 이모지 답장
→ 진짜 강아지가 답장하는 느낌을 주려고!
```

### 화면 4: 마이페이지 (mypageView)

```
[크레딧 충전]     ← +20 / +50 / +100 / 직접입력
[펫 추가 +]       ← 펫 등록 페이지로 이동
[나만의 추억 저장소] ← 추억사진 업로드 + 메모
```

### 화면 5: 펫 추가 (petAddView)

```
[펫 사진 선택]
[주인 이름]  [강아지 이름]  [좋아했던 음식]
[강아지/고양이]  [생년월일]
[상세 메모]
[펫 저장] ← 크레딧 1개 차감, 기본 영상 3개 무료 생성
```

## 서버 (server/) - 식당 주방 비유

```
유저(손님)가 주문(요청)하면 → 서버(주방)가 처리해서 → 결과를 돌려줌

주소: http://localhost:5050
```

### 서버가 받을 수 있는 주문 목록

| 주문 (API) | 무슨 뜻? | 결과 |
|-----------|----------|------|
| "Google 로그인해줘" | POST /api/auth/google | 출입증(JWT) 발급 |
| "내 정보 알려줘" | GET /api/auth/me | 이름, 크레딧, 영상 목록 |
| "이 주문번호 맞아?" | POST /api/orders/verify | 맞으면 크레딧 120 충전 |
| "영상 만들어줘" | POST /api/videos/generate | 사진→AI→영상 (크레딧 20 차감) |
| "대화 답장해줘" | POST /api/chat/reply | 랜덤 이모지 답장 |

### 서버의 데이터 저장소

```
server/data/db.json 파일에 모든 정보가 저장됨

저장되는 것들:
├── users (유저 정보)
│   └── 이메일, 이름, 크레딧, 주문인증 여부
├── videos (생성된 영상)
│   └── 영상 URL, 이름, 종류
├── jobs (작업 기록)
│   └── 작업 상태 (생성중/완료/실패)
└── claimedOrders (사용된 주문번호)
    └── 주문번호별로 누가 썼는지
```

### 비밀 설정 (.env 파일)

| 설정 이름 | 무슨 뜻? |
|----------|---------|
| PORT=5050 | 서버 문 번호 |
| JWT_SECRET | 출입증 만드는 비밀 열쇠 |
| GOOGLE_CLIENT_ID | Google 로그인 연결 ID |
| VALID_ORDER_IDS | 인정되는 주문번호 목록 |
| NANO_BANANA_API_KEY | AI 이미지 생성 열쇠 |
| KLING_API_KEY | AI 영상 생성 열쇠 |
