# PetHolo 개발 세션 로그 — 2026-03-05 (세션 4)

## 작업 요약

### 1. GIF 미리보기 기능 구현

Kling AI로 영상이 생성되면 자동으로 GIF 변환 → S3 업로드 → 프론트엔드 표시.

#### 1a. GIF 서비스 (NEW)
- **파일:** `apps/api/src/services/gif.service.ts`
- ffmpeg 2-pass 팔레트 최적화 (stats_mode=diff + sierra2_4a 디더링)
- 영상 URL → GIF 변환 → S3 업로드 → URL 반환
- 프리셋:
  - `BASE_VIDEO`: 160×216px, 3초, 10fps (베이스 영상 카드용)
  - `MOTION`: 120×120px, 2초, 10fps (모션 아이콘용)
- 임시 파일 cleanup (finally 블록)

#### 1b. AI 서비스 수정
- **파일:** `apps/api/src/services/ai.service.ts`
- `_pollKlingVideoStatus()`: 영상 완료 시 GIF 자동 생성 (try/catch로 실패해도 영상은 정상 저장)
- `resolveImageToBase64()`: URL/base64 자동 판별 + URL이면 다운로드→base64 변환
- `_processStartFrameGeneration()`: 모든 이미지를 resolveImageToBase64()로 처리

#### 1c. 프로필 서비스 수정
- **파일:** `apps/api/src/services/profile.service.ts`
- `list()` / `getById()`: 활성 베이스 영상의 gifUrl을 프로필 레벨 gifUrl로 파생

#### 1d. 세팅 페이지 GIF 표시
- **파일:** `apps/web/src/app/profiles/[id]/settings/page.tsx`
- BaseVideo/Motion 인터페이스에 `gifUrl` 추가
- API 응답 매핑에 `gifUrl` 포함
- 베이스 영상 카드: gifUrl 있으면 `<img>` 표시, 없으면 이모지 폴백
- 모션 카드: gifUrl 있으면 `<img>` 표시, 없으면 이모지 폴백
- 3분할 미니 프리뷰: 각 슬롯에 GIF 미니 이미지 표시

### 2. 프로필 사진 업로드 3개 → 1개 변경

#### 변경 이유
강아지 옷/악세서리 사진은 1장이면 충분. 여러 장 보내면 AI가 혼란.
펫 등록 시 이미 정면+전신 사진 2장을 저장하므로 재활용.

#### 2a. 프론트엔드 위자드 수정
- **파일:** `apps/web/src/app/pets/[id]/profiles/new/page.tsx`
- `petData` 상태 추가 + useEffect로 펫 정보 fetch (frontPhoto, sidePhoto)
- `clothingPhotos[3]` → `outfitPhoto` 단일 상태로 변경
- Step 2 UI:
  - 상단: 펫 정면+전신 사진 미리보기 (읽기 전용, "등록 사진" 라벨)
  - 하단: 의상 사진 1개 업로드 슬롯
- `handleGenerateImages()`:
  - `faceImage: petData.frontPhoto` (URL 그대로 전송)
  - `bodyImage: petData.sidePhoto` (URL 그대로 전송)
  - `outfitImage: outfitPhoto` (base64)
- 펫 사진 없으면 에러 처리 → 펫 상세 페이지로 리다이렉트

#### 2b. 백엔드 URL→base64 변환 (ai.service.ts에 포함)
- `resolveImageToBase64()` 헬퍼: URL이면 axios 다운로드→base64, base64이면 그대로 반환

### 3. 홀로그램 플레이어 좌우반전 재생

- **파일:** `apps/web/src/app/player/[profileId]/page.tsx`
- 베이스 영상 + 모션 영상 모두 `transform: scaleX(-1)` CSS 적용
- 이유: 아크릴 프리즘에 스마트폰을 얹어서 재생하면 반사되므로 좌우반전 필요

### 4. 기타 수정

- **analytics.schema.ts**: `z.record(z.unknown())` → `z.record(z.string(), z.unknown())` (Zod 타입 에러 수정)
- **Dockerfile**: runner 스테이지에 `RUN apk add --no-cache libc6-compat` 추가 (ffmpeg Alpine 호환성)
- **패키지 설치**: `fluent-ffmpeg`, `@ffmpeg-installer/ffmpeg`, `@types/fluent-ffmpeg`

### 5. 목업 문서 업데이트

코드 변경사항에 맞게 목업 HTML 문서들을 동기화:

  - API 설명: faceImage/bodyImage URL 재활용 + 백엔드 URL→base64 자동 변환
