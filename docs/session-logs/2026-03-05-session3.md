### 3. 데이터베이스 스키마 검토 + 백엔드 서비스 동기화

#### 3a. Pet 서비스 — species/breed/gender/memo 필드 추가
- **pet.schema.ts:** Zod 스키마에 species/breed/gender/memo 추가
  - species: `z.enum(['DOG', 'CAT', 'OTHER'])`
  - gender: `z.enum(['MALE', 'FEMALE', 'UNKNOWN'])`
  - breed: `z.string().max(50)`
  - memo: `z.string().max(500)`
- **pet.service.ts:**
  - `create()` — 4개 필드 입력 + Firestore 저장
  - `update()` — 4개 필드 수정 (nullable 지원)
  - `list()` — 4개 필드 응답에 포함
  - `getById()` — 4개 필드 응답에 포함

#### 3b. Webhook 서비스 — 중복 방지 + 주문 추적
- **webhook.service.ts:**
  - `cafe24OrderId + cafe24ProductId`로 중복 웹훅 체크 (이미 코드 발급됐으면 스킵)
  - 코드 생성 시 `cafe24OrderId`, `cafe24ProductId`, `cafe24BuyerEmail` 저장

#### 3c. Auth 서비스 — lastLoginAt 추적
- **auth.service.ts:**
  - `login()` — 이메일/비번 로그인 시 lastLoginAt 업데이트
  - `googleAuth()` — Google 로그인 시 lastLoginAt 업데이트
  - `cafe24Auth()` — Cafe24 로그인 시 lastLoginAt 업데이트
