## 1. 펫등록 → 프로필 생성 사진 연동 수정

### 문제
- 펫 등록 시 사진 2장(정면/전신)을 업로드하지만, API가 실패하면 `demo-pet-1`이라는 고정 ID로 폴백
- 프로필 생성 페이지(`/pets/[id]/profiles/new`)에서 API로 해당 펫 데이터를 가져오려 하지만 실패
- 결과: 사진이 표시되지 않고, "AI 이미지 생성" 버튼이 비활성화

### 해결
1. **`apps/web/src/app/pets/register/page.tsx`**:
   - API 실패 시 고정 ID 대신 `demo-pet-{timestamp}` 형태의 고유 ID 생성
   - 업로드된 사진 데이터를 `localStorage`에 `petholo_pet_{petId}` 키로 저장
   ```typescript
   localStorage.setItem(`petholo_pet_${petId}`, JSON.stringify({
     name: formData.name,
     frontPhoto: frontPhoto || '',
     sidePhoto: sidePhoto || '',
     breed: formData.breed || '',
   }));
   ```

2. **`apps/web/src/app/pets/[id]/profiles/new/page.tsx`**:
   - `petsApi.get()` 실패 시 localStorage에서 폴백 데이터 로드
   ```typescript
   const stored = localStorage.getItem(`petholo_pet_${petId}`);
   if (stored) {
     const parsed = JSON.parse(stored);
     setPetData({ name: parsed.name, frontPhoto: parsed.frontPhoto, sidePhoto: parsed.sidePhoto });
   }
   ```

## 3. Dev Inspector — Claude 연동 재설계

### 배경
기존 Dev Inspector는 텍스트/스타일을 브라우저에서 직접 수정하는 방식이었으나, 실용성이 부족했음.
"클릭하면 수정할걸 내가 적어주면 니가 바로 그걸 받아서 수정까지 해줄수있도록 프로그램 만들어줘야지"라는 요구에 따라 Claude 연동 방식으로 전면 재설계.

### 새로운 아키텍처
```
브라우저에서 요소 클릭
  → 하단 패널에 요소 정보 표시 (컴포넌트명, 파일 경로, 라인)
  → 수정 요청 텍스트 입력
  → "Claude에게 요청" 버튼 클릭
  → POST /api/dev/request → .claude/dev-requests.json에 저장
  → Claude가 파일 읽고 수정 실행
```

### 수정된 파일들

#### `apps/web/src/stores/useDevStore.ts` (전면 재작성)
```typescript
interface DevRequest {
  id: string;
  timestamp: string;
  element: {
    tagName: string;
    componentName: string;
    className: string;
    textContent: string;
    currentUrl: string;
  };
  source: { filePath: string; lineNumber: number; } | null;
  request: string;
  status: 'pending' | 'done';
}
```

#### `apps/web/src/components/dev/DevInspector.tsx` (전면 재작성)
- 기존: 오른쪽 사이드 패널 (320px), 3탭 (정보/텍스트/스타일)
- 변경: 하단 패널, 컴팩트 요소 정보 + 수정 요청 textarea + "Claude에게 요청" 버튼
- Ctrl+Enter로 전송, textarea 자동 포커스

#### `apps/api/src/routes/dev.routes.ts` (확장)
새 엔드포인트 추가:
- `POST /api/dev/request` — 수정 요청을 `.claude/dev-requests.json`에 저장
- `GET /api/dev/requests` — pending 상태 요청 조회
- `POST /api/dev/requests/done` — 요청 완료 처리
