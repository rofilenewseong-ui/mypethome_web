04. AI Messenger

PetHolo AI Messenger -- 반려동물이 이모지로 답장하는 감성 메신저.
펫은 오직 2-3개의 이모지 조합으로만 응답하며, 5-30분의 랜덤 딜레이로 감성적 몰입감을 제공합니다.

5. 핵심 설계 원칙 (Design Principles)

AI Messenger의 핵심 원칙과 기술적 제약 사항.

💬 응답 형식 Core

펫은 오직 이모지 2-3개 조합으로만 응답합니다. 텍스트 응답은 없으며, 이를 통해 실제 동물과 대화하는 듯한 감성을 만듭니다.

🐾🌳💕 🐶🍦😋 😺🌙 💕🌈🐶
⏳ 떼레이 메커니즘 Immersion

응답 떼레이: 5~30분 랜덤.
즉시 응답은 채팅봇처럼 느께지만, 떼레이는 "무지개다리 너머"라는 세계관과 어울리며 감성적 몰입감을 높입니다.

Min 5분 / Max 30분
🧠 AI 컨텍스트 분석 AI

AI는 펫 프로필에 저장된 컨텍스트 변수를 활용하여 이모지를 선택합니다. 이름, 좋아하던 간식, 산책 장소, 성격 등을 반영합니다.

🐷 이름 🍦 간식 🌳 산책 💕 성격
🔔 푸시 알림 Notify

떼레이 후 응답이 완료되면 푸시 알림을 전송합니다. 알림을 탭하면 바로 채팅화면으로 이동하여 펫의 응답을 확인합니다.

FCM (Firebase Cloud Messaging) 사용

7. 기술 사양 (Technical Spec)

AI Messenger 기능의 상세 기술 사양.

메시지 형식 (Message Format)
항목 사양 설명
사용자 메시지 text (한국어) 일반 텍스트 메시지, 최대 500자
펫 응답 emoji[] (2-3개) 오직 이모지 조합으로만 응답 (텍스트 없음)
타임스탬프 Firestore Timestamp 서버 타임스탬프 기준
읽음 상태 boolean 사용자 메시지에만 표시
메시지 ID auto-generated Firestore document ID
떼레이 범위 (Delay Range)
항목 값 설명
최소 떼레이 5분 (300s) 가장 빠른 응답 시간
최대 떼레이 30분 (1800s) 가장 늘은 응답 시간
분포 Gaussian (μ=15, σ=5) 15분 근처에 집중된 정규 분포
타이머 예약 Cloud Tasks 떼레이 후 함수 실행 예약
타이핑 인디케이터 떼레이 종료 30초 전 응답 직전 타이핑 애니메이션 표시
이모지 선택 로직 (Emoji Selection Logic)
사용자 메시지 의도 이모지 선택 규칙 예시
산책/외출 발바닥 + 장소 + 감정 🐾🌳💕
간식/음식 펫 얼굴 + 간식명 + 감정 🐶🍦😋
그리움/보고싶음 하트 + 무지개 + 펫 얼굴 💕🌈🐶
날씨/환경 태양/달 + 풍 + 발바닥 ☀️🌸🐾
놀이/활동 발바닥 + 활동 + 감정 🐾⚽😄
인사/안부 펫 얼굴 + 손 + 하트 🐶👋💕
컨텍스트 변수 (Context Variables)
변수명 타입 예시 용도
petName string "콩이" 펫 이름
petType string "dog" 동물 종류 (이모지 풀 결정)
petEmoji string "🐕" 대표 이모지
favoriteSnack string "고구마" 간식 관련 이모지 선택에 활용
walkingPlace string "한강 공원" 산책/외출 관련 이모지 선택에 활용
personality string[] ["활발", "애교"] 감정 이모지 선택에 활용
memorialDate Timestamp 2024-06-15 기일 관련 특별 이모지 트리거
데이터 저장소 (Storage)
항목 상세 설명
저장소 Firestore 메시지 문서 저장
경로 users/{uid}/pets/{petId}/messages 사용자 > 펫 > 메시지 계층 구조
인덱스 createdAt DESC 시간 역순 정렬 (최신 메시지 먼저)
페이지네이션 20건/페이지 무한 스크롤 로딩
실시간 구독 onSnapshot() Firestore 실시간 리스너

8. Firestore 문서 구조

메시지 저장에 사용되는 Firestore 문서 스키마.

PATH users/{uid}/pets/{petId}/messages/{messageId}
메시지 문서 경로
{
  "id": "auto-generated",
  "type": "user" | "pet",
  "content": "콩아, 오늘 산책가고 싶다",  // type=user: 텍스트
  "emojis": ["🐾", "🌳", "💕"],          // type=pet: 이모지 배열
  "isRead": true,
  "createdAt": Timestamp,
  "metadata": {
    "delaySeconds": 720,              // 적용된 떼레이 (초)
    "intentDetected": "walk",         // AI가 감지한 의도
    "contextUsed": ["walkingPlace", "personality"]  // 활용된 컨텍스트 변수
  }
}
PATH users/{uid}/pets/{petId}/messenger_state
메신저 상태 문서 (대기 상태, 마지막 읽음 등)
{
  "lastMessageAt": Timestamp,
  "pendingReply": {
    "scheduledAt": Timestamp,         // 응답 예정 시간
    "messageRef": "messages/{id}",    // 대상 메시지 참조
    "delaySeconds": 720
  },
  "unreadCount": 2,
  "lastPetEmojis": ["🐾", "🌳", "💕"]
}

9. API 엔드포인트

메신저 기능에 필요한 서버 API.

POST /api/messenger/send
사용자 메시지 전송 → AI 처리 → 떼레이 예약
GET /api/messenger/{petId}/messages
메시지 목록 조회 (페이지네이션 지원)
GET /api/messenger/{petId}/state
메신저 상태 조회 (대기 상태, 읽지 않은 수)
POST /api/messenger/{petId}/read
메시지 읽음 처리
GET /api/messenger/pets
펫 목록 + 마지막 메시지 미리보기 (선택 화면용)
