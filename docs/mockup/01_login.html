01. 카페24 진입 & 온보딩

3. Specifications

인증 방식, 등급 시스템, 기술 사양 상세 테이블입니다.

3-1. 인증 사양 (Auth Specification)

항목 상세
인증 방식 카페24 SSO (토큰 기반)
진입점 카페24 자사몰 내 링크 클릭
별도 로그인 불필요 (자동 인증)
회원 매칭 카페24 member_id 기반 자동 매칭
신규 회원 자동 생성 (Bronze 등급, 0C)
세션 유지 JWT 토큰 (24시간), Refresh Token (7일)
온보딩 최초 진입 시 1회 표시 (3단계 캐러셀)
토큰 저장 HttpOnly Cookie + localStorage fallback

3-2. 등급 시스템 (Tier System)

항목 Silver (정품 인증 완료) Bronze (샘플 체험 모드)
진입 조건 카페24에서 풀세트 구매 자사몰 링크 진입만
UI 표시 ✓ 정품 인증 완료 샘플 체험 모드
초기 크레딧 120C 0C
AI 생성 가능 불가
홀로그램 재생 가능 불가
앱 탐색 가능 가능
등급 이름 노출 사용자에게 Silver/Bronze 명칭 숨김 — "정품 인증 완료" / "샘플 체험 모드"로 표시

3-3. SSO 엔드포인트

POST /api/auth/cafe24/callback
카페24 SSO 콜백. access_token 수신 후 member_id로 사용자 매칭/생성, JWT 발급.
GET /api/auth/me
현재 로그인 사용자 정보 조회. JWT 검증 후 프로필 반환.
POST /api/auth/refresh
Refresh Token으로 Access Token 갱신.
