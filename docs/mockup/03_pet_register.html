Design Notes & Annotations

개발 시 참고해야 할 설계 주의사항

📷 사진 요구사항 & AI 생성 품질

정면 사진과 측면 사진은 AI 홀로그램 생성의 핵심 소스입니다. 정면 사진에서 얼굴 특징(눈, 코, 입, 귀 형태)을 분석하고, 측면 사진에서 체형과 비율을 파악합니다. 사진 품질이 낮거나 배경이 복잡하면 생성 결과가 부정확할 수 있습니다. 최소 512x512px, 권장 1024x1024px 이상입니다.

🎬 프로필 타입과 베이스 영상

프로필 타입(STANDING / SITTING)은 AI가 생성하는 베이스 영상의 기본 포즈를 결정합니다. STANDING: 서서 걷거나 서있는 전신 영상, SITTING: 앉아서 고개를 돌리거나 꼬리를 흔드는 영상. 선택한 타입에 맞는 베이스 비디오 템플릿 위에 사용자의 펫 이미지를 합성합니다. 한 펫에 대해 여러 프로필(자세별)을 만들 수 있습니다.

💰 크레딧 차감 플로우

크레딧은 AI 생성 요청 시 즉시 차감됩니다. 생성 실패(API 타임아웃, 서버 오류 등) 시에는 자동으로 크레딧이 환불됩니다. 사용자가 생성 버튼을 누르기 전, 차감될 크레딧 양과 현재 잔액을 명확하게 표시하여 인지된 상태에서 결정할 수 있도록 합니다. 잔액 부족 시 "크레딧 충전" 버튼으로 스토어 페이지로 이동합니다.

💬 AI 메신저 맥락 데이터

선택사항 필드(좋아하는 간식, 산책 장소, 함께 했던 이야기)는 AI 메신저 대화에서 반려동물의 성격과 추억을 반영한 자연스러운 대화를 생성하는 데 활용됩니다. 예: "초코가 좋아하던 닭가슴살 간식 기억나?" 같은 대화 맥락을 AI가 생성할 수 있습니다. 이 데이터는 추후 펫 관리 화면에서 수정할 수 있습니다.

Related API Endpoints

이 플로우에서 호출되는 주요 API

POST /api/pets

반려동물 등록 (multipart/form-data: frontPhoto, sidePhoto, name, birthDate, passingDate, breed, favoriteSnack, walkingPlace, memo)

POST /api/profiles

프로필 생성 (petId, baseType: STANDING | SITTING)

POST /api/ai/startframe/generate

스타트 프레임 AI 이미지 3장 생성 (profileId, faceImage, bodyImage, outfitImage) — 10C
faceImage/bodyImage = 펫 등록 사진 URL 재활용 (백엔드에서 URL→base64 자동 변환), outfitImage = 의상 사진 base64

POST /api/ai/startframe/select

생성된 3장 중 1장 선택 (jobId, selectedIndex)

POST /api/ai/kling/generate

Kling 베이스 영상 생성 (profileId, baseVideoId, imageUrl) — 40C

GET /api/ai/jobs/:jobId/status

AI 작업 상태 조회 (polling: 3초 간격)

GET /api/credits/balance

현재 크레딧 잔액 조회
