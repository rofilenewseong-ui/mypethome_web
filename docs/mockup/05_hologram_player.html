05. Hologram Player

홀로그램 프리즘을 위한 3분할 영상 재생기. 프로필별 베이스 영상 + 좌/우 모션 영상을 배치하고 가로 모드에서 동시 재생합니다.
아크릴 프리즘(10cm x 10cm)에 최적화된 화면 구성으로, 실물 크기의 3D 홀로그램 효과를 제공합니다.

Detailed Annotations

홀로그램 플레이어의 핵심 동작 원리와 기술적 상세 사항

◆ 3분할 레이아웃 & 프리즘 매핑

  • 3분할 화면이 아크릴 프리즘(10cm x 10cm)의 각 면에 반사됩니다
  • CENTER(45%): 정면 베이스 영상 → 프리즘 정면 반사
  • LEFT(27.5%): 좌측 모션 영상 → 프리즘 좌측 반사
  • RIGHT(27.5%): 우측 모션 영상 → 프리즘 우측 반사
  • 검은 배경(#000)은 프리즘에서 투명하게 보여 3D 부유 효과 연출

🔁 가로 모드 자동 전환

  • 재생 버튼 탭 시 자동으로 Landscape 모드로 전환
  • 시스템 화면 회전 잠금과 무관하게 강제 전환 (API 활용)
  • 뒤로가기 시 Portrait 모드로 자동 복귀
  • 상단 UI는 3초 후 자동 숨김 (탭 시 다시 표시)

☟ 롱프레스 모션 변경

  • 재생 중 LEFT/RIGHT 영역을 1.5초 이상 롱프레스
  • 진동 피드백과 함께 모션 선택 오버레이 표시
  • 영상 재생은 중단되지 않고 모션만 교체
  • CENTER(베이스 영상)은 교체 불가 — 고정

☀ 화면 밝기 & 최적화

  • 재생 시작 시 화면 밝기 자동 MAX (프리즘 투영 선명도)
  • 재생 종료 시 이전 밝기로 자동 복원
  • 화면 자동 꺼짐 방지 (WakeLock API)
  • 배터리 20% 이하 시 경고 표시

↺ 영상 루프 재생

  • 3개 영상 모두 독립적으로 무한 반복 재생
  • 각 영상의 길이가 달라도 개별 루프 처리
  • 시작 타이밍 동기화: 최초 재생 시 3개 동시 시작
  • 오디오 없음 — 시각적 추모에 집중

📷 영상 포맷 & 스펙

  • 포맷: MP4 (H.264) / WebM (VP9) 지원
  • 해상도: 768 x 768px (정사각형, 각 영상)
  • 프레임: 30 FPS 고정
  • 길이: 2~8초 (모션별 상이, 개별 루프)
  • 배경: 투명 또는 순수 검정(#000)

Technical Specifications

홀로그램 플레이어 기술 사양 상세

항목 사양 비고
영상 포맷 MP4 (H.264) / WebM (VP9) iOS Safari = MP4, Android Chrome = WebM 우선
영상 해상도 768 × 768px (정사각형) 각 슬롯 동일 해상도, Retina 대응
프레임 레이트 30 FPS 부드러운 모션 + 배터리 효율 균형
영상 길이 2~8초 (개별 루프) 모션 복잡도에 따라 상이

화면 방향 재생 시 Landscape 강제 전환 Screen Orientation API 사용
영상 좌우반전 CSS transform: scaleX(-1) 아크릴 프리즘 반사 보정 — 스마트폰을 아크릴에 얹어 재생하면 프리즘에서 반사되므로 영상을 좌우반전하여 올바른 방향으로 표시
화면 밝기 재생 시 자동 최대 밝기 Screen Brightness API / Native Bridge

제스처 — 롱프레스 좌/우 영역 1.5초 롱프레스 모션 교체 시 진동 피드백 (Vibration API)
제스처 — 탭 화면 아무곳 탭 UI 오버레이 토글 (3초 자동 숨김)
제스처 — 뒤로가기 좌상단 버튼 또는 시스템 뒤로가기 Portrait 복귀 + 밝기 복원
배경색 #000000 (순수 검정) OLED 픽셀 OFF = 완전 투명 효과
영상 동기화 3영상 동시 시작, 개별 루프 requestAnimationFrame 기반 동기

Implementation Notes

개발 참고 사항

📱 플랫폼별 고려사항

  • iOS: Screen Orientation API 미지원 → CSS transform rotate + 풀스크린으로 대체
  • Android: Screen Orientation API lock('landscape') 사용
  • PWA: manifest.json orientation 설정 활용 가능
  • 밝기 제어: Native Bridge (React Native / Capacitor) 필요

⚡ 성능 최적화

  • 3개 <video> 태그 동시 재생 — GPU 가속 필수
  • will-change: transform으로 레이어 분리
  • 저사양 기기: 해상도 512x512로 자동 다운스케일
  • 메모리 관리: 미사용 모션 영상 URL.revokeObjectURL()
