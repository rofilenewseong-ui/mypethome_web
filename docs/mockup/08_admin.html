Admin 권한 및 설정 스펙

관리자 접근 권한, 2FA 요구사항, 로그 보존 정책, 레이트 리밋 설정.

항목 설명 값 / 정책
Admin 접근 조건 Firestore users/{uid}.isAdmin === true isAdmin flag 필수
2FA 인증 TOTP 기반 2단계 인증 (Google Authenticator 호환) 모든 Admin 접근 시 필수
2FA 미인증 시 대시보드 전체 접근 차단, 2FA 설정 페이지로 리디렉트 접근 불가 (HTTP 403)
Admin 역할 SUPER_ADMIN: 전체 권한, MODERATOR: 사용자/코드 관리만 2단계 역할 구분
레이아웃 데스크톱 전용 (min-width: 1024px), 모바일 레이아웃 미지원 Desktop Only
감사 로그 보존 모든 로그 90일 보관, 이후 Cold Storage 이관 90일 (Hot) / 무제한 (Cold)
감사 로그 불변성 Firestore 보안 규칙에서 update/delete 차단, Cloud Functions에서만 write Immutable (수정/삭제 불가)
크레딧 조정 권한 SUPER_ADMIN만 가능, 사유(reason) 필수 입력, 감사 로그 자동 기록 SUPER_ADMIN + reason 필수
사용자 삭제 Soft-delete 방식, 30일 후 Cloud Function에서 hard-delete 수행 30일 유예 기간
코드 생성 제한 1회 최대 100개, 1일 최대 500개 100/batch, 500/day
API Rate Limit Admin API 요청 제한 100 req/min per admin
AI 서비스 모니터링 NanoBanana / Kling AI 상태 폴링 주기 30초 간격 health check
자동 재시도 정책 AI 서비스 실패 시 최대 3회 재시도, 3회 실패 시 크레딧 자동 환불 3회 retry, 실패 시 refund
세션 타임아웃 Admin 세션 비활동 자동 만료 30분 (idle timeout)
