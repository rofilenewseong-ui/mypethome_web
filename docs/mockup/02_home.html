02. Home - 홈 화면

PetHolo 메인 홈 화면의 모든 상태를 보여주는 목업입니다.
반려동물 미등록 상태(Empty), 단일 펫 등록 상태, 프로필 목록 확장, 다중 펫 관리 상태를 포함합니다.

Component Specification

홈 화면의 주요 구성요소와 제한사항을 정의합니다.

Component Element Description Constraints
TopBar 앱 타이틀 / 설정 / 크레딧 "PetHolo" 로고 텍스트, 설정(gear) 아이콘, 잔여 크레딧 표시 배지. Sticky 헤더로 스크롤 시 항상 노출. 크레딧 항상 표시
BottomNav 하단 네비게이션 (4 탭) 홈 / 메신저 / 플레이어 / 설정. 현재 활성 탭은 accent-blue 색상으로 하이라이트. 4개 고정 탭
EmptyState 반려동물 미등록 상태 등록된 반려동물이 없을 때 표시. 큰 "+" 버튼으로 반려동물 등록 유도. 최초 진입 기본 상태
PetCard 반려동물 카드 이모지 아바타, 이름, 품종, 기일 표시. 프로필 수 배지 포함. 탭 시 프로필 목록 확장. 이모지 아바타 필수
QuickActions 빠른 액션 버튼 (3개) 메신저 (대화하기), 프로필관리 (영상/모션 관리), 홀로그램 (재생) 바로가기. 펫 1마리 이상 시 노출
ProfileList 프로필 목록 각 프로필: 타입 배지, 썸네일, 기본영상 수, 모션 수 표시. 활성 프로필은 별 아이콘. MAX 3 base / 12 motions
PetList 다중 반려동물 목록 2마리 이상 등록 시 수평 카드 리스트 형태. 각 카드에 이모지, 이름, 프로필 수 표시. 등록 수 제한 없음
CreditDisplay 크레딧 잔액 표시 TopBar 내 배지 형태 + 펫카드 하단 충전 유도 영역. 크레딧 구매 페이지 연결. 전역 항상 표시
PetEmoji 반려동물 이모지 등록 시 이모지 선택: 강아지류(🐕🐩🦮🐕‍🦺), 고양이류(🐈🐈‍⬛🐱), 기타(🐇🐹🐦🐢🐟) 등록 시 1회 선택

User Flow

홈 화면에서의 사용자 주요 동선을 보여줍니다.

신규 사용자 앱 실행 → 홈 (Empty) → "+" 버튼 탭 → 반려동물 등록
기존 사용자 앱 실행 → 홈 (With Pet) → 어디로? → 메신저 / 프로필관리 / 홀로그램
프로필 관리 홈 (With Pet) → 카드 확장 → 프로필 목록 → 선택 / 추가 → 프로필 상세
다중 펫 홈 (Multi Pet) → 펫 선택 → 홈 (With Pet) → 어디로?
크레딧 어디서든 → 크레딧 배지 탭 → 크레딧 충전 화면

1. Home - Empty State (반려동물 미등록)

사용자에게 반려동물 등록을 유도하는 CTA가 중앙에 위치합니다.

Design Notes & Rules

홈 화면 구현 시 참고할 디자인 규칙과 제한사항입니다.

📍 Bottom Navigation
4 Tab Structure:
🏠 홈 - 메인 홈 화면 (현재 페이지)
💬 메신저 - AI 메신저 대화
▶ 플레이어 - 홀로그램 재생
⚙ 설정 - 앱 설정

Active tab: var(--accent-blue)
Inactive tab: var(--text-muted)
Position: absolute bottom, z-index: 100
Background: gradient transparent to bg-primary
💰 Credit System
항상 표시 위치:
1. TopBar 우측 - 크레딟 배지 (골드 테마)
2. 홈 화면 - 크레딟 영역 (충전 버튼 포함)

크레딟 소모:
- 메신저 대화: 1C / 메시지
- 모션 생성: 10C / 모션
- 홀로그램 재생: 무료

Format: nnnC (e.g., "120C")
⚠ Profile Limits
프로필당 제한:
🎥 기본영상: 최대 3개
🏃 모션: 최대 12개

프로필 타입:
STANDING (서있는자세) - 파란색 배지
SITTING (앉은자세) - 핑크색 배지

반려동물 수: 제한 없음
프로필 수: 제한 없음
