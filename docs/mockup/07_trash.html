Annotations / Design Notes

휴지통 기능의 핵심 설계 결정 사항 및 기술 노트

⏰ 30일 보관 정책 Retention
· 삭제 시점부터 30일 카운트다운 시작
· 자동 정리: 서버 Cron Job (매일 자정 KST)
· 0 0 * * * Asia/Seoul
· D-0 도달 시 자동 영구 삭제 실행
💰 복원 비용 구조 Credit
· 삭제 시 원래 비용의 50% 크레딧 환불
· 복원 시 원래 결제 크레딧 전액 재차감
· 잔액 부족 시 크레딧 충전 페이지 이동
· 영구 삭제 시 환불 크레딧은 유지
🔴 D-day 시각적 긴급도 UX
· D-30 ~ D-8: ● 초록 (안전)
· D-7 ~ D-4: ● 주황 (주의)
· D-3 ~ D-1: ● 빨강 (위험, 펄스 애니메이션)
· D-0: 자동 삭제 (표시 없음)

🔒 영구 삭제 불가역 Policy
· 영구 삭제는 되돌릴 수 없음
· 이중 확인 모달로 실수 방지
· 스토리지에서 즉시 파일 제거
· 관련 메타데이터 soft-delete 마킹
🕒 Cron Job 스케줄 Backend
· 매일 자정 KST (00:00 Asia/Seoul)
· expired_at ≤ NOW() 조건 조회
· 배치 처리로 S3/R2 파일 삭제
· 삭제 로그 기록 (audit trail)

API Endpoints

휴지통 관련 백엔드 API 엔드포인트 정의

GET /api/trash
휴지통 항목 목록 조회 (필터: type, 정렬: expired_at ASC)
POST /api/trash/:id/restore
항목 복원 - 크레딧 잔액 확인 후 재결제 처리, 원래 위치로 복원
DELETE /api/trash/:id
영구 삭제 - S3/R2 오브젝트 삭제 + DB soft-delete 마킹
POST /api/cron/trash-cleanup
자동 정리 Cron - expired_at ≤ NOW() 조건의 항목 배치 영구삭제 (매일 00:00 KST)

DB Schema - trash_items

휴지통 테이블 스키마 참조

CREATE TABLE trash_items (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       UUID NOT NULL REFERENCES users(id),
  item_type     TEXT NOT NULL CHECK (item_type IN ('base_video', 'motion', 'profile')),
  item_id       UUID NOT NULL,
  item_name     TEXT NOT NULL,
  original_price INTEGER NOT NULL,         -- 원래 결제 크레딧
  refund_amount  INTEGER NOT NULL,         -- 삭제 시 환불액 (50%)
  restore_cost   INTEGER NOT NULL,         -- 복원 비용 (= original_price)
  deleted_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  expired_at    TIMESTAMPTZ NOT NULL,      -- deleted_at + 30 days
  permanently_deleted BOOLEAN DEFAULT FALSE,
  restored      BOOLEAN DEFAULT FALSE,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  updated_at    TIMESTAMPTZ DEFAULT NOW()
);

-- Cron Job 조회용 인덱스
CREATE INDEX idx_trash_expired ON trash_items (expired_at)
  WHERE permanently_deleted = FALSE AND restored = FALSE;

-- 사용자별 조회 인덱스
CREATE INDEX idx_trash_user ON trash_items (user_id, item_type)
  WHERE permanently_deleted = FALSE AND restored = FALSE;
